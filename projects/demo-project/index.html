Demo Project — 示例项目

一个展示架构思路与技术选型的演示项目，涵盖前后端分离、CI/CD 流水线及容器化部署实践。

Java 21 Spring Boot 3 React 18 TypeScript Docker GitHub Actions MySQL 8 Redis

项目背景

这个项目的目标是搭建一套可复用的全栈项目脚手架，涵盖从本地开发到生产部署的完整链路。很多时候启动一个新项目时，基础设施搭建会占去大量时间，这套模板希望解决这个问题。

架构设计

项目采用前后端分离架构：

  • 后端：Spring Boot 3 + Java 21，提供 RESTful API，通过 JWT 鉴权
  • 前端：React 18 + TypeScript + Vite，通过 Nginx 反向代理访问后端
  • 数据层：MySQL 存储持久化数据，Redis 处理缓存与会话
  • 容器化：Docker Compose 编排本地环境，生产环境对接 Kubernetes

CI/CD 流水线

使用 GitHub Actions 实现自动化：

  1. Push 到 feature 分支 → 触发单元测试与 lint 检查
  2. PR 合并到 main → 触发集成测试 + Docker 镜像构建
  3. 打 Tag → 自动发布到生产环境

本地运行

# 克隆仓库
git clone https://github.com/aarontang/demo-project.git
cd demo-project

# 启动依赖服务
docker compose up -d mysql redis

# 启动后端
./mvnw spring-boot:run

# 启动前端
cd frontend && npm install && npm run dev

后续计划

目前项目还在持续迭代，计划增加的功能包括：可观测性接入（Prometheus + Grafana）、多租户支持以及更完善的权限管理模型。
